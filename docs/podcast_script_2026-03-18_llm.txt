When researchers analyze microbiome data to find differences in microbial abundance between conditions, they typically turn to specialized statistical methods. These include approaches like DESeq2 and edgeR, which model count data using the negative binomial distribution, and others like ALDEx2, ANCOM-BC2, and metagenomeSeq, which attempt to correct for the compositional nature of microbiome data where the total sum of counts is fixed.

A new study systematically evaluated eight such differential abundance testing methods using six publicly available microbiome datasets. The researchers wanted to know: when there truly is no difference between conditions, do these methods produce p-values that behave as expected?

To test this, they employed four different permutation strategies to generate datasets where the null hypothesis is known to be true. These included shuffling sample labels, shuffling counts within samples, shuffling counts within taxa, and fully randomizing the entire counts table. Under the true null, p-values should follow a uniform distribution - meaning if you sort them, they should be evenly spaced from 0 to 1.

The results were striking. Methods based on the negative binomial distribution - DESeq2 and edgeR - consistently produced p-values that were too small, indicating inflated significance. Similarly, methods designed to handle compositional data - ALDEx2, ANCOM-BC2, and metagenomeSeq - also showed this same problematic pattern of p-value inflation.

In contrast, classical non-parametric tests like the t-test and Wilcoxon test performed as expected, producing appropriately calibrated p-values that matched the theoretical uniform distribution under the null hypothesis.

This finding is particularly concerning because it suggests that widely-used, specialized microbiome analysis tools may be systematically producing false positives. The study, published on bioRxiv, highlights a fundamental issue in microbiome differential abundance analysis that could affect the reliability of many published findings in this field.

A hierarchy-aware smoothing algorithm uses flanking sequence context to recover specificity lost to multi-matching or novel k-mers. This context-aware approach allows HKS to maintain accuracy even when exact matches aren't available, by leveraging the hierarchical relationships between features.
